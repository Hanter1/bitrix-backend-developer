# Тестовое задание для backend-разработчика (Bitrix)  
**Мини-приложение: управление бонусными баллами пользователей**

## Функциональные требования

1. У каждого авторизованного пользователя есть бонусный баланс (виртуальные баллы).  
2. На главной странице приложения выводится текущий бонусный баланс.  
3. Реализовать форму для перевода баллов другому пользователю:
   - Поля: **Email получателя**, **Количество баллов для перевода**  
   - Кнопка **“Перевести”**  
4. Перевод возможен только если баланс отправителя достаточно большой (нельзя уйти в минус).  
5. После перевода создается запись в таблице истории переводов: кто перевел, кому, сколько и когда.  
6. Под формой выводится таблица с историей 10 последних переводов текущего пользователя, включая постраничную навигацию.  
7. При попытке перевода на несуществующий Email или превышении баланса — выводить соответствующую ошибку.  

## Технические требования

1. Приложение должно быть сделано на одной из младших редакций 1C-Bitrix (“Стандарт” или “Малый бизнес”).  
2. Создать отдельный шаблон сайта для тестового приложения.  
3. Весь сайт закрыт авторизацией (регистрация и восстановление пароля не требуются).  
4. Данные о пользователях и переводах хранятся в отдельных таблицах БД через **D7 ORM**.  
5. История переводов должна поддерживать постраничную навигацию.  
6. Код приложения версионируется через **git**.  
7. Для фронтенда можно использовать любой CSS/JS фреймворк, но дизайн не обязателен — основное внимание на бэкенд и логику работы.
